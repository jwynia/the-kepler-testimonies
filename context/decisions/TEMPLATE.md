## Notes for Contributors

### When to Create a Decision Record

Create one for:
- Any Tier 2 (Community Vote) decision
- Any Tier 3 (Director Decision) decision
- Any Creative Decision that significantly affects the story
- Any Technical Decision about the project's infrastructure or process

Skip for:
- Routine work that doesn't change direction
- Small bug fixes or documentation updates
- Day-to-day drafting choices

### Filename Format

Use the format: `###-[slug].md` where:
- `###` is the decision number (001, 002, 003, etc.)
- `[slug]` is a short descriptive name in lowercase with hyphens

Examples:
- `001-premise-selection.md`
- `002-governance-adoption.md`
- `003-kepler-type-vote.md`

### Finding the Right Number

Check the [Decision Index](index.md) to see what numbers are already used. Use the next available number.
